    Frontend reference¶

    The frontend is the user interface. It is plain HTML, CSS and JavaScript (no framework), using Chart.js for charts and Lucide for icons. It is served by the same FastAPI process as the API, so everything is on one origin.

    Screens¶

    The interface is a set of pages under frontend/public:

    • Login / Register - account access.
    • Scan (home) - upload one or more photos of a single receipt (drag-and-drop), with a progress indicator for upload, OCR, extraction and result.
    • Edit - the original receipt image on one side, the recognized data (merchant, date, amount, items) on the other, editable before saving; it also checks that item amounts add up to the total.
    • Calendar - a monthly view with the number of scans and the total per day, a sortable list of all receipts, and folder management.
    • Statistics - dashboards: monthly and daily charts, categories, top shops, a weekly trend and a heat map.
    • About - a static team page.

    How it talks to the server¶

    All backend communication goes through frontend/js/api.js, a single global object named SS. It attaches the session cookie to every request, handles the CSRF token (reads it from a cookie, sends it in a header for data-changing requests), redirects to the login page when the server reports the user is unauthenticated, and exposes one function per operation.

    Languages and theme¶

    The UI supports Polish and English, switchable from the navigation bar, plus a dark mode toggle. Both preferences are stored in localStorage.

    SS API client - full reference¶

    The descriptions below come from the JSDoc comments in frontend/js/api.js.

    State properties¶

    Property Type Description
    SS.apiBase string Base URL for all API requests, derived from the current origin.
    SS.currentUser Object | null The authenticated user object, or null when logged out.
    SS.currentLanguage "PL" | "EN" The active UI language, persisted in localStorage.
    SS.authenticated boolean | null Auth state: true logged in, false logged out, null not yet known.

    Session & state¶

    Method Signature Description
    user() SS.user() -> Object\|null Return the current in-memory user object.
    isAuthed() SS.isAuthed() -> boolean Whether the user is known to be authenticated.
    setSession(user) SS.setSession(user) Store the user and mark the session as authenticated.
    clearSession() SS.clearSession() Clear the user and mark the session as logged out.
    csrfToken() SS.csrfToken() -> string\|null Read the CSRF token from the cookie.

    Core request helper¶

    Method Signature Description
    api(path, opts) async SS.api(path, opts = {}) Low-level fetch wrapper: prefixes apiBase, attaches the CSRF header on writes, and redirects to login on 401. All other methods build on it.

    Authentication¶

    Method Signature Description
    login(email, password) async SS.login(email, password) Log in and store the session.
    register(username, email, password) async SS.register(username, email, password) Register a new account and log in.
    me() async SS.me() Fetch the current user's profile and refresh the session.
    ensureAuth() async SS.ensureAuth() -> boolean Confirm authentication on page load (checks the cookie via me() if needed).
    logout() async SS.logout() Log out and clear the session.
    guard() async SS.guard() Redirect to login if the user is not authenticated.
    guestOnly() async SS.guestOnly() Redirect away from guest-only pages when already logged in.

    Receipts¶

    Method Signature Description
    uploadReceipt(files) async SS.uploadReceipt(files) Upload one or more images of a single receipt for processing.
    listReceipts(params) async SS.listReceipts(params = {}) List the user's receipts (optionally filtered).
    getReceipt(id) async SS.getReceipt(id) Fetch a single receipt with its items.
    updateReceipt(id, payload) async SS.updateReceipt(id, payload) Update a receipt (merchant, total, items, importance...).
    deleteReceipt(id) async SS.deleteReceipt(id) Delete a receipt and its images.

    Folders¶

    Method Signature Description
    listFolders() async SS.listFolders() List the user's folders.
    createFolder(name) async SS.createFolder(name) Create a new folder.
    updateFolder(folderId, description) async SS.updateFolder(folderId, description) Update a folder's description.
    deleteFolder(folderId) async SS.deleteFolder(folderId) Delete a folder.
    addReceiptToFolder(folderId, receiptId) async SS.addReceiptToFolder(folderId, receiptId) Link a receipt to a folder.
    removeReceiptFromFolder(folderId, receiptId) async SS.removeReceiptFromFolder(folderId, receiptId) Unlink a receipt from a folder.

    Analytics¶

    Method Signature Description
    getDashboard(period_type) async SS.getDashboard(period_type = "monthly") Fetch aggregated dashboard data for a period (monthly, weekly, yearly).

    UI¶

    Method Signature Description
    setLanguage(lang) SS.setLanguage(lang) Set the UI language ("PL" or "EN") and persist it.
    navTranslations() SS.navTranslations() -> Object Return the navigation label translations for the active language.
    paintNav(active) async SS.paintNav(active) Render the navigation bar, marking the active page.
