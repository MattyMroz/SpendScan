    Getting started¶

    These are the exact steps to run SpendScan locally. The commands are PowerShell (Windows); on Linux use the shell equivalents.

    Prerequisites¶

    • Windows or Linux, Python 3.13+
    • NVIDIA GPU with CUDA (PaddleOCR-VL Q8 needs ~3 GB VRAM)
    • Docker Desktop (for the PostgreSQL container)
    • uv - the Python package manager (install)
    • A Gemini API key (SPENDSCAN_GEMINI_API_KEY) from aistudio.google.com

    1. Install dependencies¶

    uv sync

    2. Download llama.cpp + the OCR model (~1.4 GB)¶

    uv run python scripts/prepare_ocr_runtime.py

    This downloads into external/bin/llama/ and external/models/ocr/paddle-ocr/.

    3. Start PostgreSQL in Docker¶

    docker run --name spendscan-postgres `
  -e POSTGRES_USER=postgres `
  -e POSTGRES_PASSWORD=postgres `
  -e POSTGRES_DB=spendscan `
  -p 5432:5432 `
  -d postgres:18

    4. Load the schema¶

    docker cp backend/spendscan/db/migrations spendscan-postgres:/tmp/migrations
docker exec spendscan-postgres bash -lc "set -e; for f in /tmp/migrations/*.sql; do psql -U postgres -d spendscan -v ON_ERROR_STOP=1 -f \"\$f\"; done"

    5. Configure environment variables¶

    copy .env.example .env

    Fill in at least:

    Variable Purpose
    SPENDSCAN_GEMINI_API_KEY Your Gemini API key (data extraction).
    SPENDSCAN_JWT_SECRET Any long random string (signs the session JWT).
    SPENDSCAN_DATABASE_URL postgresql+psycopg://postgres:postgres@localhost:5432/spendscan

    For the full list of settings and their defaults, see the Configuration reference.

    6. Start the backend¶

    $env:PYTHONPATH="backend"
uv run uvicorn spendscan.api.app:app --reload

    The first start takes a few seconds to preload the OCR model into VRAM.

    7. Open the application¶

    What URL
    Frontend (login, scan, calendar, statistics) http://127.0.0.1:8000/
    Interactive API docs (Swagger) http://127.0.0.1:8000/docs
    Database health check http://127.0.0.1:8000/health/db

    To use it from a phone on the same Wi-Fi, open http://<LAPTOP_IP>:8000/ (find the IP with ipconfig).

    Local HTTP vs. production HTTPS

    Locally the app runs over HTTP, so SPENDSCAN_AUTH_COOKIE_SECURE=false is the development setting. Behind HTTPS in production, set SPENDSCAN_AUTH_COOKIE_SECURE=true.

    Development commands¶

    uv run pytest tests/unit -q     # run the unit tests
uv run ruff check .             # lint
uv run ruff format .            # format

    Reset the database (drop + recreate from the SQLModel metadata):

    $env:PYTHONPATH="backend"
uv run python scripts/reset_db.py

    Optional OCR smoke test (writes to workspace/output/smoke/):

    $env:PYTHONPATH="backend"
uv run python scripts/smoke_paddle.py

    Key API endpoints¶

    The full, interactive list is at /docs. The most important routes:

    Method Path What it does
    POST /api/v1/auth/register Register a user, set the session cookie.
    POST /api/v1/auth/login Log in, set the session cookie.
    POST /api/v1/auth/logout Log out, clear the session cookies.
    GET /api/v1/auth/me The logged-in user's profile.
    POST /api/v1/receipts Upload receipt photos -> OCR -> LLM -> save.
    POST /api/v1/receipts/batch Upload several receipts at once.
    GET /api/v1/receipts List the user's receipts.
    GET /api/v1/receipts/{id} Receipt details (items, discounts, raw OCR).
    PATCH /api/v1/receipts/{id} Edit a receipt.
    DELETE /api/v1/receipts/{id} Delete a receipt and its images.
    GET /api/v1/analytics/dashboard Aggregated dashboard data.
    GET /health/db Check the PostgreSQL connection.

    These routes are documented in full, with their handler functions, on the API layer reference.
